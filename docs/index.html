Channel-Based Architecture

Each channel creates a Matrix room, a persistent AI session, and an isolated workspace directory. Context is preserved across restarts. Conversations never bleed between channels.

!create research → room + session + workspace/research/
☇

Webhook Automation

Every channel exposes an HTTP endpoint. Wire up cron jobs, CI/CD pipelines, or external services to inject prompts programmatically.

POST /webhook/session/:id
⏱

Smart Scheduling

Timezone-aware cron and natural language scheduling. Run prompts on a timer, export and import schedule configs.

Pluggable agent backends

Swap between AI runtimes without changing your workflow. Each backend connects to the same Matrix interface.

Mux
type = "mux"

Native Rust backend using mux-rs. Streaming responses, built-in file operations, and native MCP server integration. No external binaries needed.

  • Streaming token output
  • Native file read/write/edit/bash tools
  • MCP server chaining
  • SQLite session persistence
  • Zero external dependencies
ACP
type = "acp"

Agent Client Protocol integration. Communicates with claude-code-acp or codex-acp as long-lived subprocesses. Full Claude Code ecosystem compatibility.

  • Claude Code CLI compatible
  • Long-lived warm sessions
  • ACP protocol tool use
  • Codex support
Direct
type = "direct"

Direct Claude CLI subprocess invocation. Lightweight option for simpler use cases where you want to call the CLI per-prompt.

  • Minimal setup
  • Per-prompt invocation
  • CLI flag pass-through
  • Session resume support

Meet DISPATCH█

Your orchestration agent that runs in your DM. Monitor all workspace rooms, send tasks to specific channels, and get notified when agents need your attention.

  • ⇨

    Task Dispatch

    Send prompts to any workspace room asynchronously and poll for results.

  • ◉

    Room Monitoring

    See status of all active rooms, pending tasks, and error states at a glance.

  • ⚡

    Event Notifications

    Get alerted when agents complete tasks, encounter errors, or have questions.

  • ⚙

    Admin Operations

    Create rooms, reset sessions, and manage schedules from a single DM thread.

Deploy anywhere

Single binary, Docker, or multi-tenant — gorp runs wherever you need it.

Docker (Single Instance)

The fastest path to production. All MCP tools pre-installed.

# Build and launch
docker compose up -d

# Check logs
docker compose logs -f

Docker (Multi-Tenant)

Run up to 10 isolated instances, each with their own config and workspace.

# Each instance gets its own port
docker compose -f docker-compose.multi.yml \
  up -d gorp-1 gorp-2 gorp-3

From Source

Build the release binary directly. Supports optional GUI and admin panel features.

cargo build --release
./target/release/gorp start

macOS App

Desktop GUI with system tray, hotkeys, and native notifications.

# Build .app bundle + DMG
./packaging/macos/build-app.sh
./packaging/macos/build-dmg.sh
